## How it works

1. **Define commands** using `defineCommand()` to create named actions
2. **Register keybinds** that map key combinations to command names
3. **Handle commands** using `on()` to respond when commands are triggered

```typescript
import { defineCommand, on, Key, type Keybind } from '@woven-canvas/core'

// Define a command
const MyCommand = defineCommand('my-command')

// Register keybind
const keybinds: Keybind[] = [
  { command: 'my-command', key: Key.K, mod: true }
]

// Handle command
on(ctx, MyCommand, (ctx) => {
  console.log('My command triggered!')
})
```

## Key APIs

- `defineCommand(name)` - Create a named command
- `on(ctx, command, handler)` - Register a command handler
- `Key.*` - Key constants (A-Z, 0-9, arrows, etc.)
- `Keybind` type - `{ command, key, mod?, shift? }`

## Keybind options

| Option | Type | Description |
|--------|------|-------------|
| `command` | `string` | Name of the command to trigger |
| `key` | `number` | Key constant (e.g., `Key.A`) |
| `mod` | `boolean` | Require Cmd (Mac) / Ctrl (Win) |
| `shift` | `boolean` | Require Shift key |

## Disable default keybinds

To use only your custom keybinds without plugin defaults:

```vue
<WovenCanvas :editor="{ keybinds, omitPluginKeybinds: true }" />
```
