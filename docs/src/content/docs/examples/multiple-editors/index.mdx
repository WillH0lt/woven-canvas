## How it works

Each `WovenCanvas` component creates an independent editor instance:

- Separate ECS worlds with their own entities
- Independent tool states and selections
- Isolated event handling

```vue
<template>
  <div class="grid">
    <WovenCanvas @ready="handleReady1" />
    <WovenCanvas @ready="handleReady2" />
    <WovenCanvas @ready="handleReady3" />
    <WovenCanvas @ready="handleReady4" />
  </div>
</template>
```

## Separate persistence

Use different document IDs for each editor:

```vue
<WovenCanvas :store="{ persistence: { documentId: 'doc-1' } }" />
<WovenCanvas :store="{ persistence: { documentId: 'doc-2' } }" />
```

## Sharing data between editors

To copy content between editors, export from one and import to another:

```typescript
const editor1Ref = ref<Editor | null>(null)
const editor2Ref = ref<Editor | null>(null)

function copyToEditor2() {
  // Export from editor 1
  let snapshot: any
  editor1Ref.value?.nextTick((ctx) => {
    snapshot = exportEntities(ctx)
  })

  // Import to editor 2
  editor2Ref.value?.nextTick((ctx) => {
    importEntities(ctx, snapshot)
  })
}
```

## Use cases

- **Comparison views** - Side-by-side canvas editing
- **Template galleries** - Preview multiple templates
- **Multi-page documents** - Each page is a separate canvas
- **Split editing** - Work on related content simultaneously
